## Context
Minor GAP / consistency finding from the 2026-06-01 audit (LG-07). The membership subscribe surface
has **two paths that nothing documents**, so the native-SDK path looks orphaned from the web side and
risks being mistaken for dead code and deleted.

- `MembershipController.Subscribe` ‚Üí `CreateMembershipSubscription` is the **native-SDK** flow
  (SetupIntent + EphemeralKey, see `CreateMembershipSubscription.cs:79-114`). It is wired on both the
  web customer API (`src/Cleansia.Web.Customer/Controllers/MembershipController.cs:15-25`) and the
  mobile customer API (`src/Cleansia.Web.Mobile.Customer/Controllers/MembershipController.cs:15-25`).
- The **web** `MembershipFacade`
  (`src/Cleansia.App/libs/cleansia-customer-features/profile/src/lib/membership/membership.facade.ts`)
  only calls `createCheckoutSession`, `getMine`, `getPlans`, `cancel`, `swapPlan` ‚Äî it **never** calls
  `subscribe`. Web subscribes via Stripe-hosted Checkout (`createCheckoutSession`, facade `:122-153`);
  mobile uses the PaymentSheet/SetupIntent `Subscribe` path. This split is legitimate but is recorded
  nowhere.
- The finding also pins this handler to two tracked consistency debts: **B5** (failure construction
  uses `nameof(Command)` instead of the offending field ‚Äî `consistency.md:64-66`, real call sites at
  `CreateMembershipSubscription.cs:47,61`) and **B8** (Stripe-with-no-try/catch ‚Äî `consistency.md:75-79`).

This ticket **depends_on T-0111** (LG-SEC-02, mobile direct-subscribe idempotency) and rebases on the
post-fix `CreateMembershipSubscription.cs`; do not start until T-0111 is `done`.

- [ ] **AC2 (B5 closed ‚Äî failure errors name the offending field)** ‚Äî Given the post-T-0111 handler,
  When `CreateMembershipSubscription.Handler` returns a `BusinessResult.Failure`, Then the first
  `Error` arg is `nameof` of the **offending field/entity**, not `nameof(Command)` ‚Äî i.e. the
  `BusinessErrorMessage.UserNotFound` failure (`CreateMembershipSubscription.cs:47`) no longer uses
  `nameof(Command)`, per `consistency.md` B5 (`consistency.md:64-66`). The existing per-field error
  keys (`MembershipPlanNotFound`, `MembershipAlreadyActive`) are preserved.

  subscribe and an unchanged mobile SetupIntent subscribe, When each runs after this ticket, Then both
  behave exactly as before T-0179 (no endpoint added/removed, no DTO field renamed, idempotency from
  T-0111 intact). This is a documentation + B5-rename ticket, not a behavior change.
